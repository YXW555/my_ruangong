# 环境变量配置说明

## 高德地图API Key配置

### 1. 创建环境变量文件

在 `frontend` 目录下创建 `.env` 文件（如果不存在）：

```bash
# Windows PowerShell
cd frontend
New-Item -Path .env -ItemType File
```

### 2. 配置API Key

在 `.env` 文件中添加以下内容：

```
VUE_APP_AMAP_KEY=你的高德地图API_Key
```

**注意：**
- 将 `你的高德地图API_Key` 替换为你在高德开放平台申请的API Key
- 不要使用引号包裹API Key
- 不要在等号两边添加空格

### 3. 示例

```
VUE_APP_AMAP_KEY=b2e5855e71acf8f3156a6409b5247086
```

### 4. 重启开发服务器

配置环境变量后，需要**重启前端开发服务器**才能生效：

```bash
# 停止当前服务器（Ctrl+C）
# 然后重新启动
npm run serve
```

### 5. 验证配置

启动后，打开浏览器控制台，如果看到 "高德地图API加载成功" 的日志，说明配置成功。

如果看到 "高德地图API Key未配置" 的警告，请检查：
1. `.env` 文件是否在 `frontend` 目录下
2. 环境变量名称是否为 `VUE_APP_AMAP_KEY`（必须以 `VUE_APP_` 开头）
3. 是否重启了开发服务器

## 注意事项

1. **`.env` 文件不要提交到Git**
   - 确保 `.env` 在 `.gitignore` 中
   - 可以创建 `.env.example` 作为模板

2. **环境变量命名规则**
   - Vue CLI要求环境变量必须以 `VUE_APP_` 开头
   - 只有以 `VUE_APP_` 开头的变量才会暴露给前端代码

3. **不同环境**
   - `.env` - 所有环境
   - `.env.local` - 本地环境（会被git忽略）
   - `.env.development` - 开发环境
   - `.env.production` - 生产环境
